# Research Report: compare Cursor and Windsurf

> MVP note: this report may use mock or live web sources depending on the selected search mode. Verify fast-changing facts before external use.

## Executive Summary

- Cursor 更偏向在熟悉的代码编辑器体验中嵌入 AI，适合希望保留现有编辑器习惯、同时获得代码库问答和多文件修改能力的开发者。 [S1] [S2]
- Windsurf 更强调 agentic coding workspace，让助手围绕任务保持上下文、推动从意图到实现的流程。 [S3] [S4]
- 两者核心差异可以概括为：Cursor 更像熟悉编辑器中的 AI 加速层，Windsurf 更像以 AI 协作为中心的任务工作区。 [S2] [S4]

## Research Plan

- **Cursor 和 Windsurf 的产品定位分别是什么？** 产品定位决定最终报告的主线。
- **两者在开发工作流和 AI 交互方式上有什么差异？** 这是 AI coding tool 对比中最重要的使用体验维度。
- **两者各自更适合哪些用户或团队？** 报告需要给出可行动的选择建议。
- **使用这类工具时有哪些共同风险或限制？** 避免只写优点，体现研究完整性。

## Key Claims

- **Cursor positioning:** Cursor 更偏向在熟悉的代码编辑器体验中嵌入 AI，适合希望保留现有编辑器习惯、同时获得代码库问答和多文件修改能力的开发者。 [S1] [S2]
- **Windsurf positioning:** Windsurf 更强调 agentic coding workspace，让助手围绕任务保持上下文、推动从意图到实现的流程。 [S3] [S4]
- **Workflow difference:** 两者核心差异可以概括为：Cursor 更像熟悉编辑器中的 AI 加速层，Windsurf 更像以 AI 协作为中心的任务工作区。 [S2] [S4]
- **Evaluation axes:** 评估 AI coding tools 时，应同时比较编辑器熟悉度、自动化程度、代码库上下文、协作治理、价格和模型选择。 [S5]
- **Cursor evidence:** 检索到的网页资料包含 Cursor 相关信息，可作为后续产品定位和功能对比的证据基础。 [S1] [S2]
- **Windsurf evidence:** 检索到的网页资料包含 Windsurf 或 Codeium/Windsurf 相关信息，可用于分析其 AI coding workflow。 [S3] [S4]
- **Comparison basis:** 当前资料同时覆盖 Cursor 和 Windsurf，报告可以围绕产品定位、工作流、适用人群和风险展开对比。 [S1] [S3]

## Evidence Extracts

- [S1] Cursor is an AI-first code editor built around coding workflows.
- [S2] Cursor is strongest for developers who want AI assistance without leaving a familiar editor model.
- [S3] Windsurf is presented as an AI coding environment focused on agentic software development.
- [S4] Windsurf is strongest when the user wants the coding assistant to drive larger task flows, maintain task context, and suggest next steps.
- [S5] AI coding tools are commonly compared across editor familiarity, autonomous task execution, repository context, collaboration controls, price, model options, and enterprise governance.
